# 家庭助手 — 移动端会话页面

这是一个轻量的前端示例，提供移动端友好的对话界面，并且支持通过设置面板配置自定义 API 地址和 API Key。

文件说明
- `index.html`：主页面，包含聊天区域、输入框与设置面板。
- `styles.css`：移动优先的样式（消息气泡、固定输入栏、模态设置面板）。
- `app.js`：前端逻辑，负责保存/加载配置（localStorage）、渲染消息、发送请求到配置的 API，支持本地 mock 回退。

快速开始
1. 在浏览器中打开 `index.html`（直接双击或启动简单 HTTP 服务器以避免某些浏览器的本地文件限制）。
2. 默认启用 mock 模式（设置->使用本地 mock），可直接测试发送/接收。mock 模拟延迟并返回示例文本。
3. 如果要连接真实 API：
   - 点击右上角设置按钮。填入 `API 地址`（例如：你的代理或后端接口），可选填写 `API Key` 和 `模型`。
   - 关闭设置并发送消息。

关于 API 要求
- 前端会以 POST JSON 的形式发送：{ "prompt": "用户输入", "model": "..." }
- 返回可以是 JSON 或纯文本。脚本会尝试读取常见字段 `output`, `text`, `message`，或者直接把 JSON 序列化为字符串显示。

安全提示
- 不要把真实的 API Key 提交到公共仓库。前端直接使用 Key 有泄露风险，推荐在生产中通过后端代理来隐藏 Key、做速率限制和审计。

扩展建议（可选）
- 在后端实现一个代理路由，将前端请求转发到真实的 AI 服务并注入 Key。
- 增加会话历史、持久化消息和简单的会话上下文管理（将 message 列表发给后端）。
- 增加语音输入/输出、文件/图片上传支持。
